/*
檔案名稱: CSIE_41343101_1031_2_optimized.java
功能說明: 讀取整數長度 n，接著讀入 n 個整數，將它們升序排序，並輸出成一行，各數字以空格分隔。
作者: (您可填入教師姓名)
日期: 2025-11-12
*/
import java.util.Arrays;
import java.util.Scanner;

public class CSIE_41343101_1031_2_optimized {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n;
        if (!sc.hasNextInt()) {
            sc.close();
            return;
        }
        n = sc.nextInt();
        int[] arr = new int[n];
        for (int i = 0; i < n; i++) {
            if (sc.hasNextInt()) {
                arr[i] = sc.nextInt();
            } else {
                arr[i] = 0; // 若輸入不足時，以預設值補齊，避免例外
            }
        }

        // 使用 Arrays.sort 確保正確且效率良好，處理負數也沒問題
        Arrays.sort(arr);

        // 輸出格式：同一列、數字間以空格分隔，無尾隨空格
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < n; i++) {
            if (i > 0) sb.append(' ');
            sb.append(arr[i]);
        }
        System.out.println(sb.toString());

        sc.close();
    }
}

修改原因與重點說明
- 正確性提升：改用 Arrays.sort 直接對整數陣列升序排序，確保正確且穩定的結果；原本的排序邏輯錯誤且會覆蓋原值。
- 輸出格式與穩定性：以單行輸出、數字間以空格分隔，且無尾隨空格，滿足題目輸出格式要求。處理負數天然支援。
- 程式可讀性與維護性：加入標頭註解，使用有意義的檔名與類別名稱，並使用 StringBuilder 產出字串提高效率與可讀性；並在必要時關閉 Scanner，避免資源洩漏。
- 魚戲：若輸入長度 n 與實際數字數不符，程式會以預設值填補，避免例外情況，但建議實務上可加入嚴謹的輸入驗證。
