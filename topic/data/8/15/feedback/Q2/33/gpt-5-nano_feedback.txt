改進建議：採用後綴最大值法將時間複雜度降至 O(n) 並補上適當的程式標頭註解與清晰變數命名。

改寫說明與重點修改：
- 將原本的 O(n^2) 內層搜尋改為使用後綴最大值陣列，降低時間複雜度至 O(n)。
- 加入適當的檔案標頭註解，提升可讀性與可維護性。
- 使用更清晰的變數命名與輸出方式，並在輸出前關閉 Scanner，避免資源洩漏。
- 仍符合題意：將每個元素替換為其右側元素的最大值，最後一個元素改為 -1，最後輸出整個結果。

修改後完整程式碼（Java）：
/*
 * File: ArrayRightMax.java
 * Description: 將陣列中每個元素替換為其右側元素的最大值，最後一個為 -1；並輸出結果。
 * Author: 作業提交者（請填寫姓名）
 * Date: 2025-11-14
 * Usage: 編譯執行，依題意輸入資料
 */

import java.util.Scanner;

public class ArrayRightMax {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n;
        if (sc.hasNextInt()) {
            n = sc.nextInt();
        } else {
            sc.close();
            return;
        }

        int[] arr = new int[n];
        for (int i = 0; i < n; i++) {
            if (sc.hasNextInt()) {
                arr[i] = sc.nextInt();
            } else {
                arr[i] = 0; // 預設，實務上應該避免此情況
            }
        }
        sc.close();

        // 計算每個位置右邊的最大值（不包含該位置）
        int[] rightMax = new int[n];
        if (n > 0) {
            int maxRight = arr[n - 1];
            for (int i = n - 2; i >= 0; i--) {
                rightMax[i] = maxRight;
                if (arr[i] > maxRight) {
                    maxRight = arr[i];
                }
            }
            // 最後一個元素改為 -1
            arr[n - 1] = -1;
            // 將原陣列的前 n-1 個位置更新為 rightMax
            for (int i = 0; i < n - 1; i++) {
                arr[i] = rightMax[i];
            }
        }

        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < n; i++) {
            if (i > 0) sb.append(' ');
            sb.append(arr[i]);
        }
        System.out.println(sb.toString());
    }
}
