/*
 * 檔案名稱：CSIE_41343208_1031_1.java
 * 功能：建立任意長度的一維整數陣列，讀入長度與元素，輸出升序排序結果
 * 作者：學生（修改版本）
 * 日期：YYYY-MM-DD
 * 說明：改用內建排序 Arrays.sort，避免自寫排序可能的錯誤與效率問題；加入對 n <= 0 的情況處理與完整註解。
 */

import java.util.Arrays;
import java.util.Scanner;

public class CSIE_41343208_1031_1 {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n;
        if (!sc.hasNextInt()) {
            sc.close();
            return;
        }
        n = sc.nextInt();

        if (n <= 0) {
            // 輸出一個空行，符合若無元素的情境
            System.out.println();
            sc.close();
            return;
        }

        int[] arr = new int[n];
        for (int i = 0; i < n; i++) {
            if (sc.hasNextInt()) {
                arr[i] = sc.nextInt();
            } else {
                arr[i] = 0; // 預設值，遇到缺失輸入時不致於崩潰
            }
        }

        // 使用內建排序提高效率與穩定性
        Arrays.sort(arr);

        // 使用 StringBuilder 建立輸出字串，避免尾端多餘空白
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < n; i++) {
            if (i > 0) sb.append(' ');
            sb.append(arr[i]);
        }

        System.out.println(sb.toString());

        sc.close();
    }
}

修改說明與原因：
- 將原本的選擇排序改為 Arrays.sort(…)：時間複雜度由 O(n^2) 降低到 O(n log n)，且實作更簡潔，降低出錯機率，能穩妥處理任意長度的輸入。
- 新增 n <= 0 的健全性處理：避免在 n 為 0 或負數時輸出未定義行為，改為輸出空行以符合題意。
- 輸出格式改進：改用 StringBuilder 組裝輸出，避免在最後輸出時出現多餘空白，且清晰可讀。
- 程式標頭與內部註解：新增檔案說明註解，提升可維護性與符合課程規範。

改進建議（單句）：
- 建議始用內建排序與穩健的輸入處理，以提升效率與穩定性，並在開頭加入完整的檔案註解以符合作業規範。
