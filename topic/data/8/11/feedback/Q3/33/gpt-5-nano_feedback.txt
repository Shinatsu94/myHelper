修改與說明
修改後程式碼（CSIE_41343208_1003_3.java）：
import java.util.Scanner;

public class CSIE_41343208_1003_3 {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int A = sc.nextInt();
        int B = sc.nextInt();

        // 使用 XOR 交換，僅用 A、B 兩個變數，不使用其他變數
        A = A ^ B;
        B = A ^ B;
        A = A ^ B;

        // 依題意輸出交換後的結果，先印出原本的 B，再印出原本的 A
        System.out.println(A);
        System.out.println(B);

        sc.close();
    }
}

修改原因與要點
- 直接符合題目要求：原程式與題目需求毫不相關，因此需改為讀取兩個數字並在不使用第三變數的情況下完成交換，輸出交換後的結果。
- 採用位元 XOR 交換法：能在不新增變數的前提下完成 A、B 的交換，且在 Java 的 int 範圍內安全運作。
- 輸出順序對應樣例：交換後輸出 A、B，其結果即為原本的 B、A，與給定的輸出樣例一致。
- 程式格式與註解：維持清晰縮排、具意義的變數命名，並加上註解說明交換的方式與輸出順序，同時適度關閉 Scanner。

改進建議（一句話）
若輸入數值可能超出 int 範圍，需改用長整數類型並檢視溢位風險；或改用位元操作外部更穩健的交換策略。
