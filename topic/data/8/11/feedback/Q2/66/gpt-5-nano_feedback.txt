優化後之修改內容與原因
- 新版程式遵循題意：僅讀入兩個數字，分別印出原值，然後在不使用額外變數的前提下對兩數做原地交換，最後印出交換後的值。
- 採用在 A 與 B 之間的原地交換，避免使用額外變數。此處採用異或運算（XOR）交換法，符合「不可使用其他變數進行數字交換」的限制，且不會因為兩數相加可能溢位而產生問題。
- 輸出順序與格式維持簡單直觀，符合題意：先印出 A、再印出 B，接著印出交換後的 A、B。

修改後完整程式碼與說明
- 為符合 Java 常見測試環境，使用 Class 名稱 Main，保留兩個輸入變數 A、B，並在不新增其他變數的情況下完成交換。
- 具體做法：
  - 讀入兩個整數 A、B，分別印出原值。
  - 使用 XOR 交換法直接在 A、B 兩變數上做交換，不需額外變數。
  - 印出交換後的 A、B。

修改後程式碼（完整）
public class Main {
    public static void main(String[] args) {
        java.util.Scanner scanner = new java.util.Scanner(System.in);
        int A = scanner.nextInt();
        int B = scanner.nextInt();

        // 印出原始值
        System.out.println(A);
        System.out.println(B);

        // 不使用其他變數，直接在 A、B 上以 XOR 交換
        A = A ^ B;
        B = A ^ B;
        A = A ^ B;

        // 印出交換後的值
        System.out.println(A);
        System.out.println(B);
    }
}

修改原因說明
- 目標與要求對齊：只讀入兩個數字、原地交換、不限於使用 Scanner 以外的變數。
- 選用 XOR 交換法能在不新增臨時變數的前提下完成交換，符合題意中的“不可使用其他變數進行數字交換”的限制，並且不依賴數值大小或溢位問題（相較於 A+B、A-B 方案，XOR 更安全於這個限制）。
- 保持輸入與輸出順序簡潔一致，利於自動測試。
