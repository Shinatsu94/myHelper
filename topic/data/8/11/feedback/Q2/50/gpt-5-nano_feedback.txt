修改與優化說明
- 原程式讀入三個數字並嘗試排序，與題目「讀入兩個數字並在交換後印出」不符，且輸出格式與示例不一致，故判定為不正確。
- 修改重點：
  - 僅讀入兩個數字 A, B，符合題意。
  - 使用不需要額外變數的位元運算交換法（XOR swap）將兩數互換，避免使用第三個變數，符合「不可使用其他變數進行數字交換」的要求，且可避免整數溢位風險（若改用加減法可能發生溢位）。
  - 輸出改為交換後的兩個數值，依示例輸出順序為 A 交換後的值再 B 交換後的值。
  - 增加程式註解與標頭說明，提升可讀性與維護性。

修改後完整程式碼
// 檔案名稱：SwapTwoNumbers.java
// 功能：從使用者取得兩個整數 A, B，輸出交換後的值（不使用暫存變數）
// 作者：教師
// 日期：2025-11-11

import java.util.Scanner;

public class SwapTwoNumbers {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int A = sc.nextInt();
        int B = sc.nextInt();

        // 使用 XOR 交換，避免使用額外變數
        A = A ^ B;
        B = A ^ B;
        A = A ^ B;

        System.out.println(A);
        System.out.println(B);

        sc.close();
    }
}

修改部分原因說明
- 為符合題意，必須只用 A、B 兩個變數進行交換而不引入其他變數；使用 XOR 交換法能在不額外變數的前提下完成交換，且對整數範圍適用，降低溢位風險，並與原題示例輸出一致。
