修改與改進說明
1) 核心問題與修正方向
- 問題要求：讀入兩個數字 A, B，先印出原值，再在不使用其他變數的前提下將兩數交換，並印出交換後的值。
- 原學生作答使用了三個變數（a, b, c）並以排序的方式輸出三個值，且多輸入了一個未被使用的第三數，與題意嚴格不符。
- 改用就地交換且僅使用兩個變數的方式，符合「不使用額外變數進行交換」的限制。

2) 改善要點
- 移除多餘變數，僅保留兩個變數 a, b 代表 A, B。
- 先印出原始值 a, b；再使用就地交換法（不新增變數）進行兩數交換：
  - a = a + b;
  - b = a - b;
  - a = a - b;
  這樣就能在不使用第三個變數的情況下完成交換；若對溢位有顧慮，也可以改用位元運算 XOR 交換法。
- 加入程式標頭註解、明確的變數命名，以及在結尾關閉 Scanner，提升可讀性與維護性。

3) 修改後完整程式碼
/*
 文件名稱: CSIE_41343205_1003_2.java
 功能: 讀取兩個整數 A, B，先印出原值，接著在不使用額外變數的前提下交換兩數，並印出交換後的值。
 作者: 請輸入作者名稱
 日期: 2025-11-11
*/
import java.util.Scanner;

public class CSIE_41343205_1003_2 {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int a = sc.nextInt();
        int b = sc.nextInt();

        // 印出原始值
        System.out.println(a);
        System.out.println(b);

        // 就地交換：不使用額外變數
        a = a + b;
        b = a - b;
        a = a - b;

        // 印出交換後值
        System.out.println(a);
        System.out.println(b);

        sc.close();
    }
}

修改原因說明
- 依題意，交換需在不使用第三個變數的前提下完成。原程式使用了多於兩個變數且輸入數量與題目不符，故不符合題意與要求的輸入輸出格式。
- 透過就地交換（無額外變數）滿足交換約束，同時保持輸出順序符合「先印出原值，再印出交換後的值」的要求。
- 增加標頭註解與適當註解，提升可讀性與維護性；結尾關閉 Scanner，避免資源外洩。

一句話改進建議
- 建議改用 XOR 交換法或加減法就地交換並附上輸入範圍檢查，以避免溢位風險並提升健壯性。
