## 2. 優缺點分析

### 優點
1. **判斷邏輯正確**  
   - `grade < 50 → 不及格`、`50 ≤ grade < 60 → 補考`、`grade ≥ 60 → 及格` 完全符合題目規定。  
2. **使用 `Scanner` 讀取使用者輸入**，符合 Java 基本 I/O 作法。  
3. **變數命名** (`grade`) 簡潔且具意義，易於理解。

### 缺點
| 項目 | 問題說明 |
|------|----------|
| **缺乏說明** | 程式沒有任何註解或檔案頂部說明，閱讀者不易快速了解功能。 |
| **格式不佳** | 縮排混雜、括號與關鍵字未加空格、`while` 迴圈與大括號寫在同一行，降低可讀性。 |
| **無限迴圈** | `while(true)` 會持續等待輸入，題目只要求讀入 **三筆** 分數，未在三次後結束。 |
| **例外處理** | `main` 方法宣告了 `throws InterruptedException`，但程式內根本不會拋出此例外，屬於多餘的宣告。 |
| **資源未釋放** | `Scanner` 沒有在使用
