## 2. 優缺點分析

### 優點
1. **功能完整**：能讀取三個分數並正確輸出對應等級。  
2. **基本結構正確**：使用 `for` 迴圈控制次數，`if‑else` 判斷條件正確。  
3. **程式碼可讀**：縮排一致，變數命名簡潔（`num`）。

### 缺點
1. **缺乏說明註解**：沒有說明程式的目的與使用方式。  
2. **資源管理不當**：`Scanner` 在迴圈內每次都 new，且未呼叫 `close()`，會造成資源浪費。  
3. **多餘的輸出**：`System.out.print("輸入分數: ");` 會在測試平台上產生額外文字，導致答案比對失敗。  
4. **判斷式可簡化**：`else if (num < 60 && num >= 50)` 可寫成 `else if (num >= 50)`，更易讀。
