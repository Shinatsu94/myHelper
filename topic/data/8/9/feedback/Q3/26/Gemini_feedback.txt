**3. 改進的程式碼：**

```java
import java.util.Scanner;

class ScoreCheck {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        while (sc.hasNext()) { // 修改hasNextInt()為hasNext()，處理非整數輸入
            if (sc.hasNextInt()) {
                int score = sc.nextInt();

                // 輸入驗證：檢查分數是否在有效範圍內
                if (score >= 0 && score <= 100) {
                    if (score >= 60) {
                        System.out.println("及格");
                    } else if (score >= 50) {
                        System.out.println("補考");
                    } else {
                        System.out.println("不及格");
                    }
                } else {
                    System.out.println("輸入分數超出範圍 (0-100)");
                }
            } else {
                System.out.println("輸入格式錯誤，請輸入整數。");
                sc.next(); // 消耗掉錯誤的輸入，避免無窮迴圈
            }

        }
        sc.close(); // 在程式碼結束時關閉 Scanner
    }
}
```

**4. 修改說明：**

*   **`while (sc.hasNext())`:** 將 `hasNextInt()` 改為 `hasNext()`。原因：當輸入非整數時，`hasNextInt()` 會持續返回 `false`，造成無限迴圈。改為 `hasNext()` 可以檢查是否有下一個輸入，然後再判斷是否為整數。
*   **輸入驗證：** 在讀取分數後，加入 `if (score >= 0 && score <= 100)` 判斷，確保分數在 0 到 100 之間。原因：避免處理無效分數，使程式更健壯。
*   **錯誤處理：** 當輸入不是整數時，顯示錯誤訊息並使用 `sc.next()` 消耗掉錯誤的輸入。原因：避免程式崩潰或進入無限迴圈。
*   **`sc.close();`:**  將 `sc.close()` 移到 `while` 迴圈外，確保在所有輸入處理完畢後才關閉 `Scanner`。原因：釋放資源，避免資源洩漏。
