## 2️⃣ 優缺點分析

### 優點
1. **判斷邏輯正確**：`if‑else` 的條件寫法正確，能正確映射分數到三種結果。
2. **變數命名簡潔**：`num` 能表達「分數」的意圖，符合題目要求不必改名。
3. **程式結構簡單**：只有一個迴圈與判斷，易於閱讀。

### 缺點
| 項目 | 問題描述 |
|------|----------|
| **缺少說明** | 沒有任何註解或檔案頭說明，違反「備註」項目。 |
| **主函式簽名錯誤** | `public static void main()` 不是 Java 執行入口，必須接受 `String[] args`。 |
| **缺少 import** | 使用 `Scanner` 前未 `import java.util.Scanner;`，編譯會失敗。 |
| **無限迴圈** | `while(true)` 讓程式永遠等待輸入，若使用者在最後沒有再輸入，程式不會自然結束。 |
| **資源未關閉** | `Scanner` 未在結束前 `close()`，不符合良好資源管理。 |
| **格式細節** | 少了空行、括號與關鍵字之間的空格，雖不影響功能，但影響可讀性。 |
