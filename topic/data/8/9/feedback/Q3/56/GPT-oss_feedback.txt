## 2. 優缺點分析

### 優點
1. **邏輯正確**：對於單一分數的判斷符合題目規定，`>=60`、`50~59`、`<50` 三個區間均正確輸出。
2. **基本格式**：縮排、括號使用清晰，程式結構易於閱讀。
3. **使用 `Scanner` 讀取輸入**：符合 Java 標準的輸入方式。

### 缺點
1. **缺乏說明**：沒有任何註解或檔案頭說明，無法快速了解程式目的與使用方式。
2. **只能處理單一輸入**：題目與測試案例要求一次讀入多筆成績並分別輸出結果，原程式只讀取第一個整數即結束。
3. **命名風格**：類別名稱 `ch3` 不符合 Java 的命名慣例（應為 `Ch3`），雖不影響編譯，但不夠專業。
4. **資源未關閉**：`Scanner` 建立後未呼叫 `close()`，雖在簡單程式中影響不大，但養成良好習慣仍然重要。
5. **條件冗餘**：`else if(score<60 && score>=50)` 中的 `score<60` 完全不必寫，已在前面的 `if(score>=60)` 排除。
