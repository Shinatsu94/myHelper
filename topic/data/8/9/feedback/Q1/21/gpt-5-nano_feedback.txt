修改與改進說明
- 需求重點改正：原 student 程式僅輸出 Hello World，未符合題目「印出虎科大與你的中文姓名每個字及對應內碼」的要求，因此必須輸出六行字與對應內碼。
- 優化方向與實作要點：
  - 使用字元陣列與迴圈，避免重複的 print 語句，使程式更具可維護性與可擴充性。
  - 以每個字元的 Unicode 編碼作為「內碼」，輸出十進位數值，提升可讀性與一致性。
  - 加入程式註解與標頭，提升文件可讀性與可追溯性。
  - 對變數與常量採用有意義的命名，符合基本格式規範。
- 修改後的完整程式碼如下（已符合題意：先印「虎科大」，再印「林武杰」這三個字的內碼）：
  /*
   檔案名稱: CSIE_41343119_0919_1.java
   功能: 印出「虎科大」與「林武杰」每個字及其 Unicode 十進位內碼
   作者: 教師
   日期: 2025-11-12
  */
  public class CSIE_41343119_0919_1 {
      public static void main(String[] args) {
          // 先印出「虎科大」的每一個字及其對應內碼
          String[] characters = {"虎", "科", "大", "林", "武", "杰"};
          for (String chStr : characters) {
              char ch = chStr.charAt(0);
              int codePoint = (int) ch;
              System.out.println(chStr + " " + codePoint);
          }
      }
  }

修改部分的原因
- 以迴圈與陣列取代單純多次呼叫 println，提升可維護性與可讀性；若日後要擴充名稱，只需在陣列中追加字元即可。
- 以 Unicode 編碼（十進位）輸出內碼，提供統一且常見的數值表示，避免因字型顯示造成的混淆。
- 加入清楚註解與檔案標頭，符合程式設計的基本風格與可維護性要求。

改進建議（一句話）
- 建議把「內碼」的輸出格式再與需求方確認（十進位、十六進位、或特定編碼表如 Big5/UTF-8 的位元組序列），並在程式中提供一個選項以方便切換輸出格式。
