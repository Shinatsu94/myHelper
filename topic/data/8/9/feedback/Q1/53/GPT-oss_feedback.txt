## 2. 優缺點分析

### 優點
1. **變數宣告清晰**：使用 `char` 變數分別儲存每個中文字符，易於閱讀。
2. **基本結構正確**：`public static void main(String[] args)` 及 `System.out.println` 的使用皆符合 Java 語法。
3. **縮排基本正確**：程式碼的層次結構易於辨識。

### 缺點
1. **缺乏說明**：程式未加入任何註解，無法快速了解程式目的。
2. **輸出格式錯誤**：題目要求「字 空格 代碼」一行，而程式輸出為「字:代碼」。
3. **錯誤的變數使用**：印出「大」時誤用了 `b`，導致代碼不正確。
4. **錯誤的代碼順序**：題目要求先印「虎科大」再印姓名，程式雖然順序正確，但若未來更改姓名順序，易產生混亂。
5. **變數命名可改進**：雖然符合題目需求，但 `a、b、c、d、e、f` 隱含意義不明，若加入註解或改為更具描述性的名稱，可提升可讀性。
