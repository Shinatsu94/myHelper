修改後完整程式碼（符合題意：在不額外宣告新變數的情況下，交換 A 與 B，並輸出交換後的結果）：
快照式展示：
import java.util.Scanner;
public class Main {
  public static void main(String[] args){
    Scanner scn = new Scanner(System.in);
    int A = scn.nextInt();
    int B = scn.nextInt();

    // 在不使用第三個變數的前提下，使用位元運算進行交換
    A = A ^ B;
    B = A ^ B;
    A = A ^ B;

    // 輸出交換後的結果，符合題目樣例：輸出 A 再輸出 B
    System.out.println(A);
    System.out.println(B);
  }
}

修改說明與原因：
- 直接讀入兩個整數並在不使用額外變數的情況下進行交換，符合題目「不可使用其他變數進行數字交換」的限制。
- 使用位元異或交換（A = A ^ B; B = A ^ B; A = A ^ B）避免可能的整數溢位風險，且不需要額外臨時變數，符合題目要求。
- 輸出順序設為交換後的 A 與 B（即原本的 B 與 A），以符合提供的樣例輸出（輸出 5, 3 而非 3, 5）。
- 程式結構清晰、變數命名使用 A、B，並以簡單的邏輯完成任務。

一句話改進建議：
- 如需避免同時支援極大整數範圍，可考慮檢查輸入範圍或改用其他不需額外變數的交換方法（如兩數相加再相減的做法，在極端情況下可能產生溢位，但若範圍受限則可使用）。
