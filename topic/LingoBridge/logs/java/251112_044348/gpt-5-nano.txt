修改後完整程式碼
/*
檔案名稱: LeibnizPiCalculator.java
功能說明: 使用勒侖茲級數近似圓周率 pi，輸出在 i=10000,20000,...,100000 對應的 pi 值；再找出最小的 i，使 pi 的近似值四捨五入至小數點後5位為 3.14159。
作者: 教師姓名
日期: 2025-11-11
說明: 從頭以漸進累加方式計算，避免重複計算並提升效率。
*/
public class LeibnizPiCalculator {
    public static void main(String[] args) {
        // Part (a): i 漸增輸出 i = 10000, 20000, ..., 100000 的 pi 近似值
        double sum = 0.0; // 代表 Σ_{k=0}^{i-1} (-1)^k / (2k+1)
        for (int i = 1; i <= 100000; i++) {
            // 加上第 i 個項 (-1)^(i-1) / (2*(i-1) + 1)
            double term = ((i - 1) % 2 == 0) ? (1.0 / (2.0 * (i - 1) + 1.0))
                                           : (-1.0 / (2.0 * (i - 1) + 1.0));
            sum += term;

            if (i % 10000 == 0) {
                double piApprox = 4.0 * sum;
                System.out.printf("i = %6d, pi = %.10f%n", i, piApprox);
            }
        }

        // Part (b): 找到最小的 i，使 pi 的近似值四捨五入至小數點後5位等於 3.14159
        // 使用逐步累加方式，保持 sum 的連續性
        sum = 0.0;
        int i = 0;
        while (true) {
            i++;
            double term = ((i - 1) % 2 == 0) ? (1.0 / (2.0 * (i - 1) + 1.0))
                                           : (-1.0 / (2.0 * (i - 1) + 1.0));
            sum += term;
            double piApprox = 4.0 * sum;
            if (Math.round(piApprox * 1e5) == 314159) {
                // 輸出符合題意的格式：i 以 6 位寬度顯示
                System.out.printf("i = %6d%n", i);
                break;
            }

            // 安全機制：若超過合理範圍仍未符合條件，程式自動中止以避免無限迴圈。
            // (此範例在實作中通常會在某個合理的 i 之後符合條件)
            if (i > 1000000) {
                System.out.println("未在合理範圍內找到符合條件的 i。");
                break;
            }
        }
    }
}

程式修改說明
- 由原本的雙層迴圈改為單一漸進式累加：使用 sum 變數累計前 i 項的項和，並以逐步增加的方式加入下一項，避免對每個 i 都重新從頭計算，提升執行效率。
- 變數命名與結構調整：將 T 改以 sum、piApprox 等更具語意的變數命名，讓邏輯更易於閱讀與維護。
- 內文註解加入：在程式中加入清楚的註解，說明每段邏輯的用途與輸出條件，提升可讀性。
- 程式標頭註解：新增檔案名稱、功能、作者、日期等開頭註解，符合備註部分的要求。
- 輸出格式與規範對齊：維持原題目指定的輸出格式，第一部分輸出 i=10000…100000 的 pi 值，第二部分輸出滿足條件的最小 i；對於 Part (b) 的輸出，僅輸出 i 的值，符合題目要求的輸出樣式。
- 邏輯健壯性：透過 Math.round(piApprox * 1e5) 的方式判定是否達到 3.14159 的五位小數近似值，比單純比較浮點數大小更穩定。
