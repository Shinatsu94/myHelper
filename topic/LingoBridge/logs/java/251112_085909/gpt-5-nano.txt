/*
檔案名稱: PiCalculation.java
功能說明: 計算 pi 使用莱布尼茲級數
          (a) 輸出 i = 10000, 20000, ..., 100000 對應的 pi 值，格式為
              i =  10000, pi = 3.1414926536
              ...
          (b) 找出使 pi 首次達到或超過 3.14159 的最小 i，並輸出該 i
作者: 請填寫
日期: 請填寫
*/

public class PiCalculation {
    public static void main(String[] args) {
        double sum = 0.0;
        int minimalI = -1;

        // 逐次累加 Leibniz 公式的項，直到 pi 首次>=3.14159
        for (int i = 1; ; i++) {
            sum += ((i % 2 == 1) ? 1.0 : -1.0) / (2.0 * i - 1.0);
            double pi = 4.0 * sum;

            // 輸出規格：i 顯示到 6 位數，pi 顯示到小數點後 10 位
            if (i <= 100000 && i % 10000 == 0) {
                System.out.printf("i = %6d, pi = %.10f%n", i, pi);
            }

            // 找出最小的 i 使 pi >= 3.14159
            if (pi >= 3.14159) {
                minimalI = i;
                break;
            }
        }

        // 輸出 b) 的結果：最小 i
        System.out.printf("i = %d%n", minimalI);
    }
}

修改說明與原因
- 將學生原始作業的無關邏輯（反轉整數輸出）完全移除，改為實作題目要求的 Leibniz 級數計算。
- 使用單一迴圈累加所有項，逐步更新 sum 與 pi，避免分段重複計算，提高執行效率。
- 輸出格式調整為符合題目要求：
  - i 以寬度 6 右對齊輸出（%6d），符合「i 显示到6位數」的需求。
  - pi 以浮點數輸出，且小數點後 10 位（%.10f）。
- 同時在迴圈中同時處理兩件事：(a) 當 i 為 10000、20000、...、100000 時輸出對應的 pi 值；(b) 直到 pi 首次 >= 3.14159，取得最小 i 並輸出。
- 增加程式註解與檔案頭說明，提升可讀性與維護性。
