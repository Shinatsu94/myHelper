修改與優化
1) 核心改動
- 目標：符合題意，僅使用兩個變數（A, B）進行交換，且輸出交換前的兩個值順序依題意呈現。
- 解法：使用位元異或（XOR）交換法，避免額外變數與溢位風險；交換完成後直接印出 A, B，順序為交換後的 A, B，即輸出 B, A 的結果，符合範例。

2) 修改後的完整程式碼
import java.util.Scanner;

public class CSIE_41343219_1003_2 {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int A = sc.nextInt();
        int B = sc.nextInt();

        // 使用不新增額外變數的交換方法（XOR swap）
        A = A ^ B;
        B = A ^ B;
        A = A ^ B;

        System.out.println(A);
        System.out.println(B);

        sc.close();
    }
}

修改說明與原因
- 為何改成兩個變數 A, B：符合題意“讀入兩個數字並交換後印出”，避免不必要的三數排序與第三個暫存變數（原作使用 c 作暫存），使程式更貼近題目需求。
- 為何使用 XOR 交換法：在不使用附加變數的前提下實作交換，符合題目限制。避免可能的整數溢位問題（若採用 A = A + B; B = A - B; A = A - B 的做法，可能會有溢位風險）。
- 為何輸出順序是 A 再 B：因交換後 A 代表原始的 B，B 代表原始的 A，符合題意的「交換後再印出」的輸出樣例（例如輸入 3, 5 → 輸出 5, 3）。
