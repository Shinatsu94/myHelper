修改後程式碼（Java）
```java
/*
檔案名稱: LeibnizPi.java
功能: 使用萊布尼茲公式計算 π，輸出 i = 10000, 20000, ..., 100000 對應的 pi 值，pi 為 10 位小數；並找出最小的 i，使 pi 四捨五入到小數第 5 位等於 3.14159。
作者: 你的名字
日期: 2025-11-11
*/
public class LeibnizPi {
    public static void main(String[] args) {
        double sum = 0.0;

        // 逐項累加前 100000 項
        for (int j = 0; j < 100000; j++) {
            sum += ((j % 2 == 0) ? 1.0 : -1.0) / (2.0 * j + 1.0);
            if ((j + 1) % 10000 == 0) {
                int i = j + 1;
                double pi = 4.0 * sum;
                System.out.printf("i = %6d, pi = %.10f%n", i, pi);
            }
        }

        // 空一行分隔
        System.out.println();

        // 從第 100001 項開始繼續累加，找最小的 i 使 pi 與 3.14159 的差小於 0.000005
        int iMin = -1;
        for (int j = 100000; j < 2000000; j++) {
            sum += ((j % 2 == 0) ? 1.0 : -1.0) / (2.0 * j + 1.0);
            double pi = 4.0 * sum;
            if (Math.abs(pi - 3.14159) < 0.000005) {
                iMin = j;
                break;
            }
        }
        System.out.println("i = " + iMin);
    }
}
```

修改說明與原因
- 針對題目需求，原始學生程式並未實作計算 pi 的演算法，而是做與題意完全不相關的數字反轉，且缺乏適當的標頭註解與輸出格式設定。新的版本直接實作萊布尼茲級數，符合題意。
- 輸出部分改用格式化輸出，確保 i 顯示為寬度 6 的欄位，pi 顯示到小數點後 10 位，符合題目給定的輸出格式與對齊需求。
- 演算法採用累加方式逐項增加，因此可以在同一變數 sum 上累積，先輸出 i = 10000, 20000, ..., 100000 對應的 pi，之後再延伸計算找到最小的 i 使 pi 與 3.14159 的誤差小於 0.000005，符合「四捨五入至 5 位小數為 3.14159」的理解。
- 變數命名與程式結構改為易讀：使用 sum 累計、明確的迴圈分段、以及適當的標頭註解，提升可讀性與維護性。
