/**
 * File: SwapTwoNumbers.java
 * 功能: 讀入兩個整數 A、B，印出交換後的值。不得使用臨時變數進行交換（僅使用 A、B）。
 * 作者: 你的學生（可替換為實際姓名）
 * 日期: 2025-11-11
 */
import java.util.Scanner;

public class SwapTwoNumbers {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int A = sc.nextInt();
        int B = sc.nextInt();

        // 不使用額外變數的交換（XOR 交換法）
        A = A ^ B;
        B = A ^ B;
        A = A ^ B;

        System.out.println(A);
        System.out.println(B);

        sc.close();
    }
}

修改說明與理由：
- 回歸題意：原作業要求「讀入兩個數字並印出交換後的值」，且不得使用其他變數進行交換。原程式讀入三個數字、排序後輸出，與題意不符且需要額外變數，故需重構成僅讀兩數並在不使用臨時變數的情況下完成交換。
- 交換實作：採用不用額外變數的交換方法（XOR 交換法），符合題目限制，且不易因中間值溢位（相較於 A = A + B 的做法），可在兩個整數之間完成交換。
- 標頭與註解：新增程式檔案標頭註解，說明檔名、功能與基本作者日期等，提升後續維護與閱讀性。
- 格式與命名：維持清晰的縮排與簡潔變數名稱 A、B，便於閱讀與評分。

改進建議（一句話）：
若未來需避免 XOR 相關的特殊情況，建議採用有限制的三路交換法或在輸入範圍確保不會溢位的前提下，加入註解說明該交換方法的利弊與適用範圍。
