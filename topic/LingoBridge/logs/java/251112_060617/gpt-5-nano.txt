// File: LeibnizPi.java
// 功能: 使用莱布尼茨級數計算 pi；(a) 輸出 i=10000,20000,...,100000 對應的 pi 值（保留十位小數，i 以6欄寬輸出）
//       (b) 找到最小的 i 使 pi 的近似值四捨五入到小數點後五位為 3.14159，輸出該 i
// 作者: 由教師提供的作業評分示例
// 日期: 2025-11-11

public class LeibnizPi {
    public static void main(String[] args) {
        final int MAX_I = 100000;
        double sum = 0.0;
        int sign = 1;
        int nextPrint = 10000;

        // 逐步累加莱布尼茨級數，並在指定 i 值輸出 pi 的近似值
        for (int k = 1; k <= MAX_I; k++) {
            sum += sign / (2.0 * k - 1.0);
            if (k == nextPrint) {
                double pi = 4.0 * sum;
                System.out.printf("i = %6d, pi = %.10f%n", k, pi);
                nextPrint += 10000;
            }
            sign = -sign;
        }

        // 問題 (b): 找到最小的 i 使 pi 的近似值四捨五入成 3.14159（5 位小數）
        // 判斷準則：piApprox >= 3.141585 且 piApprox < 3.141595
        sum = 0.0;
        sign = 1;
        int k = 1;
        int minIForFiveDecimals = -1;
        while (true) {
            sum += sign / (2.0 * k - 1.0);
            double piApprox = 4.0 * sum;
            if (piApprox >= 3.141585 && piApprox < 3.141595) {
                minIForFiveDecimals = k;
                break;
            }
            sign = -sign;
            k++;
            if (k > 2000000) { // 安全機制，避免無窮迴圈
                break;
            }
        }
        System.out.println("i = " + minIForFiveDecimals);
    }
}

- 從原始無關題目的程式，改為專注於題目需求的實作，讓程式能輸出指定的 i 與 pi 值，以及尋找最小的 i 以得到 3.14159 的近似。
- 採用一次性累加的方式計算莱布尼茨級數，讓整體效率更高；不用對每個 i 重新從頭計算，透過累積和與符號交替的方式逐步更新。
- 輸出格式調整為符合題目要求：
  - i 使用 %6d 的欄寬，以符合「i 顯示到6位數」的需求，並以單個換行符號分隔每行。
  - pi 使用 %.10f，顯示小數點後10位。
  - 每行輸出格式為: i = [6欄位整數], pi = [10位小數]，與題目範例對齊。
- (b) 以穩健的判斷條件求得最小的 i：
  - 使用 piApprox 在區間 [3.141585, 3.141595) 內的判斷，代表四捨五入到小數點後五位會得到 3.14159，符合題目「最小 i」的要求。
