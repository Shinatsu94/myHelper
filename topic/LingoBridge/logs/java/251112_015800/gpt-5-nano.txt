修改與優化說明
- 問題理解與實作方向
  - 原始作業僅需讀入兩個整數 A, B，並印出其值，接著交換兩者再印出。學生原始程式與需求嚴重不符，且邏輯與輸入輸出形式皆錯誤。
- 主要修改內容
  - 重新實作讀入兩個整數、分別印出原始值與交換後的值。
  - 在不使用額外變數的前提下，使用 XOR 交換法來完成兩數交換，避免整數相加造成的溢位風險。
  - 增加適當註解，並遵循較清晰的命名與格式規範。
- 修改後完整程式碼
  import java.util.Scanner;

  public class CSIE_51015130_1003_3 {
      public static void main(String[] args) {
          Scanner sc = new Scanner(System.in);
          int A = sc.nextInt();
          int B = sc.nextInt();

          // 先印出原始值
          System.out.println(A);
          System.out.println(B);

          // 不使用額外變數，使用 XOR 交換法
          A = A ^ B;
          B = A ^ B;
          A = A ^ B;

          // 再印出交換後的值
          System.out.println(A);
          System.out.println(B);

          sc.close();
      }
  }

- 修改原因說明
  - 以原本要求為基礎，先印出兩個讀入的值，再進行交換並印出；這符合「印出其值，將這二個變數的值交換並印出」的直覺順序。
  - 為避免整數溢位問題，使用 XOR 交換法，符合「不可使用其他變數進行數字交換」的限制，同時不需要額外中間變數。
  - 增加註解與合理縮排，提升可讀性與可維護性。

一句話改進建議
- 若未來要更健壯，建議加上輸入驗證與處理 extremes（例如極大或極小整數），並在需求變更時以方法化拆分交換與輸出，以提升可測試性與可重用性。

說明
- 若題目要求僅輸出交換後的兩個值而不需原始值，請把初始印出原始值的兩行移除；本實作遵循「先印出原始值、再印出交換後的值」的解題步驟。
