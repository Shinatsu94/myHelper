修改後完整程式碼與修改原因如下：

修改後程式碼
- 程式碼會先印出原始輸入值 A 與 B，接著再進行位元異或交換，最後印出交換後的值。

/**
 * Main.java
 * 功能: 讀入兩個整數 A, B，先輸出原值，再交換後輸出新值。
 * 作者: 你的名字
 * 日期: 2025-11-11
 * 說明: 依題意不使用額外變數進行交換，採用 XOR 交換法；新增程式頭註解與適當的變數命名。
 */

import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        // 以有意義的變數命名，讀入兩個整數
        int A = sc.nextInt();
        int B = sc.nextInt();

        // 先印出原始值
        System.out.println(A);
        System.out.println(B);

        // 使用 XOR 交換，不使用額外變數
        A = A ^ B;
        B = A ^ B;
        A = A ^ B;

        // 印出交換後的值
        System.out.println(A);
        System.out.println(B);

        sc.close();
    }
}

說明修改部分的原因
- 印出原始值：符合題意「印出其值」的前提，讓使用者看到初始 A, B。
- 有意義的變數命名：A, B 代表兩個變數，較易閱讀且符合題意。
- 程式頭註解：提供檔案名稱、功能、作者與日期，方便維護與評分。
- 保持單一交換邏輯且不使用額外變數：仍使用 XOR 交換法，避免額外變數的需求，符合題目限制。
- 關閉 Scanner：資源管理的好習慣，避免資源洩漏。

一句話改進建議
- 若題意確實只需要四行輸出且測資格式固定，考慮與題目需求保持一致，或在程式註解中明確說明輸出格式與測資預期，以避免混淆。
