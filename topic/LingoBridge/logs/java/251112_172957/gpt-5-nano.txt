/* 
   檔案名稱: CSIE_41343208_1031_2.java
   功能: 讀入長度 n，接著讀入 n 個整數，輸出升冪排序後的數字，支援負數。
   作者: 教師姓名
   日期: 2025-11-12
*/
import java.util.Arrays;
import java.util.Scanner;

public class CSIE_41343208_1031_2 {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        if (!sc.hasNextInt()) {
            sc.close();
            return;
        }

        int n = sc.nextInt();
        int[] arr = new int[n];

        for (int i = 0; i < n; i++) {
            if (sc.hasNextInt()) {
                arr[i] = sc.nextInt();
            } else {
                // 若輸入不足，預設為 0
                arr[i] = 0;
            }
        }

        // 使用內建排序以確保正確性、效能與對負數的支援
        Arrays.sort(arr);

        // 輸出，避免尾差的空格
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < n; i++) {
            if (i > 0) sb.append(' ');
            sb.append(arr[i]);
        }
        System.out.println(sb.toString());

        sc.close();
    }
}

修改說明與原因
- 使用 Arrays.sort 取代手寫排序邏輯
  - 原程式採用錯誤的選擇排序邏輯（每次取最大值放入前段，且最後把最後一個元素設為 -1），會產生錯誤結果且無法處理負數。
  - 使用 Java 內建排序方法 Arrays.sort 能正確、簡潔地完成升冪排序，並天然支援負數與任意長度的整數陣列，且效能穩定。
- 輸出格式處理
  - 原程式在輸出時使用特定方式輸出，且最後附加 -1，會造成輸出錯誤。
  - 改用 StringBuilder 逐步加入元素，確保輸出格式只有空格分隔，並在最後印出換行，避免尾端多餘空格。
- 程式註解與頭部說明
  - 原程式缺少檔案註解與功能說明，影響可維護性。
  - 新增完整的檔案註解與功能說明，符合「備註」需求與可讀性。
