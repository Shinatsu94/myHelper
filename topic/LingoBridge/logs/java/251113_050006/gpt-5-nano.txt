修改說明與理由
- 核心問題：學生原程式與題目「印出虎科大及學生中文姓名的每個字及其對應的內碼」無關，只做了一個讀整數並判斷及/補考/不及格的流程，未達題意。
- 修改重點：
  1) 移除不相關的輸入處理，改為直接輸出虎科大與學生姓名的每個字及其對應的內碼。
  2) 使用字元本身透過 (int)字元取得其 Unicode 編碼點，並逐字輸出，確保「內碼」以該字的實際編碼表示。
  3) 程式結構改為較規範的 Java 標準寫法：使用簡潔的主程序，並新增輔助方法 printCharsWithCode 以提高可讀性與可重用性。
  4) 加上程式註解與標頭，並使用較具可讀性的類名。
- 主要修改內容的原由：使程式符合題目要求並能正確輸出每個字的內碼，且具可維護性與可讀性。

修改後完整程式碼
文件名稱與說明可自行調整，以下為可直接執行的版本（Java 7+ 相容）：

public class PrintUnicodeCodes {
    /*
     * 檔案名稱: PrintUnicodeCodes.java
     * 功能: 印出「虎科大」與「林武杰」的每個字的內碼（Unicode 編碼點）
     * 作者: 你自己
     * 日期: 2025-11-12
     */

    public static void main(String[] args) {
        // 先印輸「虎科大」的每個字及其內碼
        printCharsWithCode("虎科大");
        // 再印學生中文姓名「林武杰」的每個字及其內碼
        printCharsWithCode("林武杰");
    }

    private static void printCharsWithCode(String s) {
        for (int i = 0; i < s.length(); i++) {
            char ch = s.charAt(i);
            System.out.println(ch + " " + (int) ch);
        }
    }
}

修改說明：
- 直接輸出每個字與對應的內碼，符合題意，避免不必要的輸入行為。
- 使用 printCharsWithCode 方法提高可維護性與可讀性，方便日後擴充其他字串。
- 字元的內碼以 Unicode 編碼點呈現，透過 (int) ch 取得，對 BMP 字元（如虎、科、大、林、武、杰）皆正確。
