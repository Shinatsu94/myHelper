修改後的完整程式碼（符合題目規範的輸出格式，計算 Leibniz 公式並輸出 i 與 pi，i 為 10000,20000,...,100000；π 以小數點後10位顯示，i 以6位數寬顯示）：
import java.util.*;

public class PiLeibniz {
    public static void main(String[] args) {
        int[] terms = {10000, 20000, 30000, 40000, 50000, 60000, 70000, 80000, 90000, 100000};
        for (int i : terms) {
            double sum = 0.0;
            for (int k = 0; k < i; k++) {
                double term = (k % 2 == 0) ? 1.0 / (2 * k + 1) : -1.0 / (2 * k + 1);
                sum += term;
            }
            double pi = 4.0 * sum;
            System.out.printf("i = %6d, pi = %.10f%n", i, pi);
        }
    }
}

說明修改部分與原因：
- 將原始提交的與題目無關的數字反轉程式，改為實作題目要求的勒盧尼茲級數計算。這個公式是 pi 的常見近似方式，逐項累加直到指定的項數 i，最後乘以 4 即可得到 pi 的近似值。
- 使用 double 以提高浮點計算精度，並依照題目要求將 pi 顯示為小數點後10位、i 顯示為6位寬，輸出格式使用 System.out.printf 以保證對齊與格式一致性。
- 對輸出格式的排版指示（兩個空格在等號後）由格式字串 "i = %6d, pi = %.10f" 產生，符合題目提供的輸出樣式。
