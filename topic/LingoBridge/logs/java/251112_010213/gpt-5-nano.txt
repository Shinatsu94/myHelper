改良與說明
- 問題重點與原始作業差異：
  - 原始程式誤把題意理解為需要三個輸入變數並以最大最小中位數排序輸出，且使用多於兩個變數進行交換，與題目「讀入兩個數字 A, B，印出原值，再交換並印出」不符，且在兩個輸入的情境會因缺少第三個輸入而失敗。
- 提出之優化思路與原因：
  - 使用者只需兩個變數 A、B，且在交換時不可另行新增變數。為符合題意，採用原地交換（不新增變數），盡量避免使用易於研判的額外儲存空間。
  - 為避免整數溢出風險，採用位元異或（XOR）交換法，僅使用 A 與 B 兩個變數完成交換，符合題意限制。
  - 程式結構清晰：先輸出原值，再輸出交換後的值，並保留程式說明的註解與適當的縮排。
- 修改內容摘要：
  - 移除多餘的第三變數 C，以及不符合題意的三數排序邏輯。
  - 使用 XOR 交換法在不新增變數的前提下完成 A 與 B 的交換，並輸出順序符合「先印出原值 A、B，再印出交換後的值 B、A」的要求。
  - 為程式加入適當的檔案註解、註解及整齊的排版。

public class CSIE_41343216_1003_2 {
    public static void main(String[] args) {
        java.util.Scanner sc = new java.util.Scanner(System.in);

        int A = sc.nextInt();
        int B = sc.nextInt();

        // 先輸出原始值
        System.out.println(A);
        System.out.println(B);

        // 不使用額外變數，利用位元異或交換 A 與 B
        A = A ^ B;
        B = A ^ B;
        A = A ^ B;

        // 再輸出交換後的值
        System.out.println(A);
        System.out.println(B);

        sc.close();
    }
}

- 移除了不必要的第三變數 C，避免偏離題意與輸入規格。
- 以 XOR 交換法在不新增變數的前提下實作交換，符合「不使用其他變數進行數字交換」的限制，並避免可能的整數溢位風險（相較於加減法交換更穩定）。
