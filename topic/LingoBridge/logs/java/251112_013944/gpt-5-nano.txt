修改後的完整程式碼（符合題意：讀入兩個整數 A、B，先印出其值，然後交換並印出）  
說明：為符合「不可使用第三方變數進行交換」的限制，採用算術交換法僅使用 A 與 B 兩個變數進行交換；先輸出初始值，再輸出交換後的結果。

import java.util.Scanner;

public class JavaSwapTwoNumbers {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        int A = scanner.nextInt();
        int B = scanner.nextInt();

        // 印出原始值
        System.out.println(A);
        System.out.println(B);

        // 使用算術運算交換兩個變數，禁止再使用其他變數
        A = A + B;
        B = A - B;
        A = A - B;

        // 印出交換後的值
        System.out.println(A);
        System.out.println(B);

        scanner.close();
    }
}

修改的部分原因說明
- 對題目需求的適配：原程式與題意完全不符，未完成兩數輸入與交換的功能，因此需要用兩個變數 A、B 讀取輸入、印出初始值、再進行交換並印出結果。
- 交換實作：依題意限制「不可使用其他變數進行數字交換」，採用算術交換法（A = A + B; B = A - B; A = A - B），只使用 A、B 兩個變數，避免引入額外的臨時變數。
- 輸出格式：依題意與範例輸出，兩個原始值與交換後的值各佔一行，共四行輸出。

改進建議（一句話）
- 為避免在極端數值時的整數溢位，建議在實務中改用長整數（long）或加入輸入驗證與錯誤處理，並考慮同時提供不使用算術交換的替代解法以提高魯棒性。
