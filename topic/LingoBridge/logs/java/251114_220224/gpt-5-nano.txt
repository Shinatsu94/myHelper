修改說明與理由：

- 使用 Arrays.sort 進行排序，時間複雜度 O(n log n)，適合任意長度陣列的排序需求。
- 考慮負數支援與輸入格式多樣性，直接逐一讀取 n 個整數。
- 增加程式標頭註解與適當的變數命名，提升可讀性與維護性。
- 使用 StringBuilder 連接輸出，避免多次輸出導致的性能問題，並避免尾端多餘空格。

以下為修改後的完整程式碼：
/**
 * File: SortingArray.java
 * 功能: 讀取整數長度 n，接著讀入 n 個整數，將陣列由小到大排序後輸出
 * 作者: your_name
 * 日期: 2025-11-14
 * 說明: 支援負數，輸出以空格分隔，最後換行
 */
import java.util.Arrays;
import java.util.Scanner;

public class SortingArray {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        int[] arr = new int[n];

        for (int i = 0; i < n; i++) {
            arr[i] = sc.nextInt();
        }
        sc.close();

        Arrays.sort(arr);

        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < n; i++) {
            if (i > 0) sb.append(' ');
            sb.append(arr[i]);
        }
        System.out.println(sb.toString());
    }
}
