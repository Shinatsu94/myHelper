修改說明與優化
- 原因與方向
  - 原程式讀入三個整數但題目僅要求讀入兩個並在交換後輸出，因此會造成讀取不足或輸出不符合題意。此外，原程式輸出順序為最小、中位、最大，與題意要求的「交換後輸出」不符。
  - 為符合題意且避免使用額外變數做交換，採用算術交換法 (a = a + b; b = a - b; a = a - b) 進行兩數交換，且只使用原有兩個變數進行交換，符合「不可使用其他變數進行數字交換」的限制。
  - 為提升穩健性，改用 long 類型以降低整數溢位風險；同時讓輸出符合題意：交換後的值以 a, b 的順序輸出，各占一行。

修改後完整程式碼
思路說明：讀入兩個數字，使用算術交換法在不額外宣告新變數的前提下完成交換，最後輸出交換後的兩個值，各自換行。

```java
import java.util.Scanner;

public class SwapTwoNumbers {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        long a = sc.nextLong();
        long b = sc.nextLong();

        // 不使用額外變數，透過算術交換法交換 a 與 b
        a = a + b;
        b = a - b;
        a = a - b;

        System.out.println(a);
        System.out.println(b);

        sc.close();
    }
}
```

修改部分說明
- 將變數型別改為 long，降低溢位風險。
- 移除不必要的額外變數與額外輸出，改用算術交換法在兩個變數上完成交換，並直接輸出交換後的兩個值，符合題意與「不使用其他變數進行交換」的限制。
- 增加註解說明交換方式的原因與注意事項，並加上適當的程式結構與關閉 Scanner。

改進建議
- 如擔心算術交換造成溢位風險，且題目允許使用位元操作且不額外變數，可考慮使用位元異或交換法（需確保兩數不相同，以避免結果相同的情況下出現問題），或改用臨時變數實作交換以提升穩定性。
