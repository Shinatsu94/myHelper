修改後的程式碼（動態計算 pi，並依題意完整輸出）
```java
/*
File: LeibnizPi.java
Description: 使用莱布尼茲級數計算 π，輸出每 10000 項時的近似值；再找出使 pi 四捨五入到小數點第 5 位為 3.14159 的最小項數。
Author: 你
Date: 2025-11-11
*/
import java.util.Locale;

public class LeibnizPi {
    public static void main(String[] args) {
        Locale.setDefault(Locale.US);

        // (a) i = 10000, 20000, ..., 100000 的 pi 近似值
        double sum = 0.0;
        int totalTerms = 100000;       // 總計算項數 (i 為項數)
        int checkpointStep = 10000;     // 每多少項輸出一次結果
        int nextCheckpoint = checkpointStep;

        for (int k = 0; k < totalTerms; k++) {
            double term = ((k % 2 == 0) ? 1.0 : -1.0) / (2.0 * k + 1.0);
            sum += term;

            if (k + 1 == nextCheckpoint) {
                double piApprox = 4.0 * sum;
                System.out.printf("i = %6d, pi = %.10f%n", k + 1, piApprox);
                nextCheckpoint += checkpointStep;
            }
        }

        // (b) 找最小 i 使 pi ≈ 3.14159（取四捨五入到小數點第 5 位為 3.14159）
        sum = 0.0;
        int i;
        for (i = 0; ; i++) {
            double term = ((i % 2 == 0) ? 1.0 : -1.0) / (2.0 * i + 1.0);
            sum += term;
            double piApprox = 4.0 * sum;

            // 四捨五入到小數點第 5 位
            if (Math.round(piApprox * 1e5) == 314159) {
                break;
            }
        }
        System.out.printf("\ni = %6d%n", i);
    }
}
```

- 移除預先硬編碼的 pi 值，改以動態計算：符合題目要求的「根據公式逐步求和」的精神，提升正確性與通用性。
- 使用 4 倍乘法得到 pi 的近似值：符合原公式 pi = 4 ∑ (-1)^k /(2k+1) 的定義，避免把結果放在未經證實的陣列中。
- 逐步輸出固定步長的結果：確保在 i = 10000, 20000, …, 100000 時輸出與題目相同的格式與數值。
- 為 (b) 設計判斷條件：以四捨五入到小數點第 5 位是否為 3.14159 作為判定標準，找出最小的 i，並符合題意輸出格式。
