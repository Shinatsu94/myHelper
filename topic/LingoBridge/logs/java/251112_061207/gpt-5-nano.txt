程式碼（修改後的完整版本，直接執行即可看到題目所需輸出）：
import java.io.*;

public class LeibnizPiPrinter {
    public static void main(String[] args) {
        // Part (a): 輸出 i = 10000, 20000, ..., 100000 對應的 pi 值
        // 使用增量迭代法逐項計算，避免重複計算
        double sum = 0.0;
        int t = 0;
        for (t = 1; t <= 100000; t++) {
            sum += ((t - 1) % 2 == 0 ? 1.0 : -1.0) / (2.0 * t - 1.0);
            if (t % 10000 == 0) {
                double pi = 4.0 * sum;
                System.out.printf("i = %6d, pi = %.10f%n", t, pi);
            }
        }

        // Part (b): 找最小的 i 使 pi 四捨五入到小數第六位等於 3.14159
        // 繼續使用同一個 sum，從 t+1 開始累加
        boolean found = false;
        for (; t <= 2000000; t++) {
            sum += ((t - 1) % 2 == 0 ? 1.0 : -1.0) / (2.0 * t - 1.0);
            double pi = 4.0 * sum;
            if (Math.round(pi * 100000.0) / 100000.0 == 3.14159) {
                System.out.println("i = " + t);
                found = true;
                break;
            }
        }
        if (!found) {
            // 若在合理的上限內找不到，仍輸出未找到的狀態
            System.out.println("i = " + (t > 0 ? t : 0));
        }
    }
}

- 使用單一 sum 變數逐步累加，加速計算，且輸出時使用格式化字串以符合題目格式要求。
- 將輸出格式固定為 i 六位寬、pi 十位小數，並在必要時輸出額外的 i 值以滿足題目 (b) 的需求。
