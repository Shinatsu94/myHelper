修改後完整程式碼：
```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        int A = scanner.nextInt();
        int B = scanner.nextInt();

        // 先印出原值
        System.out.println(A);
        System.out.println(B);

        // 不使用額外變數的交換法（只用 A, B 兩個變數）
        A = A + B;
        B = A - B;
        A = A - B;

        // 印出交換後的值
        System.out.println(A);
        System.out.println(B);
    }
}
```

修改說明：
- 移除了原作中與題意無關的邏輯機制（無意義的 count 與 digitSum 方法），專注實作題目要求：讀入兩數、各自印出、再交換並印出。
- 使用不新增變數的交換法（算術交換法），符合「不可使用其他變數進行數字交換」的限制，僅用現有的 A、B 兩個變數完成交換。
- 結構與命名改為常見的 Java 程式風格：使用 class Main、清楚的註解、可讀的變數命名，方便後續維護。

一句話改進建議：
- 為避免整數溢位風險，若可能遇到非常大的整數，建議改用位元交換法（如 A ^= B; B ^= A; A ^= B；或使用額外區塊的長整數型別）或先驗證輸入範圍。
