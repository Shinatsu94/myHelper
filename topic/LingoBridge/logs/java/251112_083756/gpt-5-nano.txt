程式碼如下（完整檔案內容）：
import java.io.*;

 /*
  File: LeibnizPi.java
  Description: 使用 Leiniz 公式計算 pi，輸出 i=10000, 20000, ..., 100000 對應的 pi 值，並找出使 pi 四捨五入到 3.14159 的最小 i。
  Author: 設計者
  Date: 2025-11-12
 */

public class LeibnizPi {
    public static void main(String[] args) {
        // Part (a): 輸出 i = 10000, 20000, ..., 100000 對應的 pi 值
        double sum = 0.0;
        int computedTerms = 0;
        for (int iTerm = 10000; iTerm <= 100000; iTerm += 10000) {
            for (int k = computedTerms; k < iTerm; k++) {
                sum += ((k & 1) == 0 ? 1.0 : -1.0) / (2.0 * k + 1.0);
            }
            computedTerms = iTerm;
            double pi = 4.0 * sum;
            System.out.printf("i = %6d, pi = %.10f%n", iTerm, pi);
        }

        // Part (b): 找出最小的 i 使 pi 四捨五入至 3.14159
        double target = 3.14159;
        int iForTarget = -1;

        // 從目前累積到 i = 100000 的 sum 繼續累加
        int k = 100000;
        while (k < 2000000) { // 設一個安全上限，避免無窮迴圈
            sum += ((k & 1) == 0 ? 1.0 : -1.0) / (2.0 * k + 1.0);
            int iTerm = k + 1;
            double pi = 4.0 * sum;
            double rounded = Math.round(pi * 100000.0) / 100000.0;
            if (rounded == target) {
                iForTarget = iTerm;
                break;
            }
            k++;
        }

        System.out.println("i = " + iForTarget);
    }
}

- 使用增量累加取代每次從頭計算：原先的思路會對每個 i 重新從 0 累加，時間複雜度較高且效能低。改用從前一階段的總和繼續累加，可以大幅提升效率，且邏輯清晰、易維護。

- 增加副功能：在同一程式中同時輸出 (a) 部分以及 (b) 部分的最小 i，提供更完整的解題能力，並包含最小 i 的輸出。
- 加入檔頭註解與較清晰命名：提升可讀性與可維護性，方便日後他人理解與修改。
