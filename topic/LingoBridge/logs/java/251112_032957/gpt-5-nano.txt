  - 程式說明註解與標頭資訊與題意不一致，影響備註分數。
- 主要改動

  - 使用不需額外臨時變數的算術交換法：A = A + B; B = A - B; A = A - B。
  - 交換完成後輸出交換後的結果，與題意的輸出樣式（兩行分別為交換後的 A 與 B）相符。
  - 保留簡單的程式結構、適度的縮排，並在必要處加入關閉 Scanner 的呼叫以釋放資源。
- 為什麼這樣修改
  - 直接符合題意：兩個數字交換後印出，且不使用額外變數進行交換，滿足作業要求。
  - 算術交換在大多數情況下簡潔且易於理解；若擔心整數溢位風險，可改用無額外變數的 XOR 交換法，但需注意二進位運算對有號整數的效果。
- 可能的進階改進

        int A = sc.nextInt();
