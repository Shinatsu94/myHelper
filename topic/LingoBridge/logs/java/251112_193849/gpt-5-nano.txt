```java
/*
檔案名稱: CSIE_41343232_1031_2.java
功能: 讀入整數數列，依序排序成升序，輸出排序後序列
作者: 學生
日期: 2025-11-12
*/
import java.util.Arrays;
import java.util.Scanner;

public class CSIE_41343232_1031_2 {
    public static void main(String[] args) {
        Scanner in = new Scanner(System.in);

        int n = in.nextInt();
        int[] arr = new int[n];

        for (int i = 0; i < n; i++) {
            arr[i] = in.nextInt();
        }

        // 使用內建排序，確保正確且簡潔
        Arrays.sort(arr);

        for (int i = 0; i < n; i++) {
            System.out.print(arr[i]);
            if (i < n - 1) System.out.print(" ");
        }
        System.out.println();

        in.close();
    }
}
```

修改說明（為何這些改動）：
- 正確性與邏輯：原程式的排序邏輯錯誤且最後還把 a[n-1] 設為 -1，結果與需求不符。改用 Arrays.sort(arr) 直接將整數陣列升序排序，確保正確性與可讀性，同時避免自訂排序邏輯帶來的錯誤。
- 格式與命名：將變數名稱改為 arr，較有意義；保留原有結構與輸出格式，維持相同輸出樣式。
- 備註：加入標頭註解，說明檔案名稱、功能、作者與日期，提升可維護性。
- 其他：保留原本的輸入方式與結束輸出方式，仍符合「一次執行結束、不使用 EOF」的要求。

一句話改進建議：
- 建議在輸入處加入基本驗證（如 n 非負且不超過合理上限），並對例外輸入做穩健處理，以提升程式的健壯性。
