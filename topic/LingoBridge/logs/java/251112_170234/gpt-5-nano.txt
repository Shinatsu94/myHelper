修改與優化說明
修改後完整程式碼（使用內建排序與更穩健的輸出方式，並適度整理資源管理與輸出格式）：
請見下方原程式的改寫版本，維持原本功能需求但提升可讀性與效率。

import java.util.Scanner;
import java.util.Arrays;

public class CSIE_41343224_1031_1 {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        int n;
        if (!input.hasNextInt()) {
            input.close();
            return;
        }
        n = input.nextInt();
        int[] array = new int[n];
        for (int i = 0; i < n; i++) {
            array[i] = input.nextInt();
        }
        input.close();

        // 使用內建排序提升效能與穩定性
        Arrays.sort(array);

        // 使用 StringBuilder 組字串，避免逐次輸出造成的成本
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < n; i++) {
            if (i > 0) sb.append(' ');
            sb.append(array[i]);
        }

        System.out.print(sb.toString());
    }
}

修改部分的原因與說明
- 使用 Arrays.sort(array) 取代自訂的選擇排序：內建排序在實作與最佳化方面通常優於手寫的簡單排序演算法，能穩定且快速地處理一般情況，且對負數與重複值皆正確。
- 使用 StringBuilder 組裝輸出字串：減少多次 I/O 呼叫的開銷，特別是在陣列元素較多時能提升執行效率，並避免尾端多餘空格的問題。
- 關閉 Scanner 以釋放資源：雖然在短暫程式執行中影響不大，但良好資源管理是良好習慣。
- 輸出格式保持原樣：數字以空格分隔，無尾部多餘空格，符合題目輸出要求。
