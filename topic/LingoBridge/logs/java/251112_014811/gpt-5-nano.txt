修改後完整程式碼
/*
File: SwapTwoNumbers.java
功能: 讀入兩個整數 A、B，印出其值，交換後再印出兩值。
作者: 教授
日期: 2025-11-11
*/
import java.util.Scanner;

public class SwapTwoNumbers {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        int A = scanner.nextInt();
        int B = scanner.nextInt();

        // 印出原始值
        System.out.println(A);
        System.out.println(B);

        // 不使用額外變數，利用加減法交換 A 與 B
        A = A + B;
        B = A - B;
        A = A - B;

        // 印出交換後的值
        System.out.println(A);
        System.out.println(B);
    }
}

修改說明與原因
- 正確性與符合度
  - 將原本與題目需求嚴重不符的補充計算邏輯與不相關變數（count、digitSum、for 迴圈等）移除，改為直接讀取兩個整數並依題意輸出兩次結果。
  - 使用不新增額外變數的交換方法 A = A + B; B = A - B; A = A - B，符合題目「不可使用其他變數進行數字交換」的限制。
- 邏輯與可讀性
  - 程式邏輯簡單清晰：先輸出原始值，再在不使用額外變數的前提下完成交換，最後輸出交換後的值。
  - 增加註解說明交換的實作方式與流程，提升可讀性。
- 格式
  - 新增了標頭註解，包含檔案名稱、功能、作者、日期等，並為程式加入必要的內部註解。
  - 變數命名 A、B 使用單字母，雖然在一般專案中可能不建議，但在教學題目中為直覺易懂，整體排版與縮排保持一致。
- 備註
  - 檔案內附上標頭註解，符合備註欄位的要求，說明檔案名稱、功能、作者與日期，以及內部註解說明。

改進建議或反饋（一句話）
- 建議在實作時先釐清題意（是否需要同時輸出原值與交換後的值），並避免引入與題目無關的邏輯與變數，以提升正確性與可維護性。
